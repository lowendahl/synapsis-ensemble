# Ensemble — Naming

## The full stack

```
Synapsis             ← future umbrella for the GUI layer (TBD trademark check)
└── Ensemble         ← this product. The workspace.
    ├── Voice        ← one role-shaped agent (brand-voice, ux-critic, etc.)
    ├── Council      ← broadcast to N selected Voices at once
    ├── Solo         ← talk to one Voice with the doc panel open
    ├── Instrument   ← a tool the Voice is allowed to call (filesystem, /docx, m365_*, etc.)
    ├── Pipeline     ← (later) a sequence of skill calls across Voices
    └── Atelier      ← (later) a layout for free-form multi-pane work
```

## Why these names

- **Ensemble** — small group of performers, each in their own register, playing the same piece. Beats "team", "panel", "swarm" — those imply hierarchy or chaos. An ensemble is *coordinated peers*.
- **Voice** — already the right word for brand-voice, copy voice, etc. Generalizes. Avoids "agent" (loaded with hype) and "assistant" (loaded with subordination).
- **Council** — what you call it when Voices vote / weigh in together. Beats "broadcast", "swarm", "team chat".
- **Instrument** — what each Voice plays. The musical metaphor stays consistent. Beats "tool" (too literal) and "skill" (Clawpilot already owns that word).

## Capitalization

- Sentence case in body copy: "Add a voice", "review with the council".
- Title Case only for proper feature names: "Council bar", "Solo mode".
- Never SCREAMING.

## Don't say

- "AI agent" (use **Voice**)
- "Multi-agent system" (use **ensemble** or **council**)
- "Prompt" in user-facing copy (use **ask** or **send**)
- "LLM" anywhere user-facing
